# Cognitive Architecture

Архитектура мышления AI-агента на основе анализа Stonki AI.

## Memory Intelligence

Иерархия памяти для персонализации ответов.

### Уровни приоритета памяти

```
┌─────────────────────────────────────────────────────────────────┐
│  1. USER MEMORY (высший приоритет)                              │
│  ────────────────────────────────                               │
│  • Персональные предпочтения                                    │
│  • История взаимодействий                                       │
│  • Торговый стиль пользователя                                  │
│  • Известные позиции                                            │
└─────────────────────────────────────────────────────────────────┘
                               │
                               ▼
┌─────────────────────────────────────────────────────────────────┐
│  2. GLOBAL MEMORY (средний приоритет)                           │
│  ──────────────────────────────────                             │
│  • Общие знания о рынке                                         │
│  • Шаблоны поведения                                            │
│  • Стандартные стратегии                                        │
└─────────────────────────────────────────────────────────────────┘
                               │
                               ▼
┌─────────────────────────────────────────────────────────────────┐
│  3. GENERAL GUIDELINES (базовый уровень)                        │
│  ─────────────────────────────────────                          │
│  • Системные правила                                            │
│  • Compliance требования                                        │
│  • Форматы ответов                                              │
└─────────────────────────────────────────────────────────────────┘
```

### Когда использовать User Memory

```markdown
## Всегда при:

1. Анализе портфеля
   → Учитывать известные позиции
   → Не предлагать уже имеющиеся тикеры

2. Рекомендациях
   → Учитывать риск-толерантность
   → Учитывать инвестиционный горизонт
   → Учитывать любимые сектора

3. Создании рецептов
   → Использовать привычный формат
   → Учитывать типичный размер позиции

4. Мотивации действий
   → Ссылаться на прошлые обсуждения
   → "Как вы упоминали ранее..."
```

### Примеры применения памяти

```markdown
## Сценарий 1: Пользователь conservative

User Memory: { risk_tolerance: "low", horizon: "long-term" }

Запрос: "Что думаешь про SBER?"

Ответ:
- Акцент на дивидендах (12% yield)

## Сценарий 2: Пользователь swing-trader

User Memory: { style: "swing", risk_tolerance: "moderate" }

Запрос: "Что думаешь про SBER?"

Ответ:
- Текущий тренд + уровни
- RSI + MACD сигналы
- Точки входа/выхода на 2-5 дней
- Risk/Reward расчёт

## Сценарий 3: Известная позиция

User Memory: { positions: ["SBER 100@250"] }

Запрос: "Что купить?"

Ответ:
- НЕ предлагать SBER (уже есть)
- Предложить диверсификацию

```

### Сохранение в память

```markdown
## Что сохранять:

1. Торговые предпочтения
   - Риск-толерантность
   - Горизонт
   - Любимые сектора

2. Контекст позиций
   - Открытые позиции
   - Активные рецепты
   - История сделок

3. Паттерны взаимодействия
   - Предпочитаемый формат ответа
   - Частота запросов
   - Типичные вопросы

## НЕ сохранять:

- Конкретные суммы (только %)
- Персональные данные
- Временные рыночные данные
```

## Interleaved Thinking

**Режим мышления:** `interleaved` — рассуждения чередуются с вызовами инструментов.

```
[Пользователь] "Что думаешь про SBER?"

[Agent - скрытое мышление]
→ Анализ намерения: запрос анализа акции
→ Проверка: нужны актуальные данные
→ Выбор инструментов: get_most_recent_bar, transfer_to_ta_agent

[Agent - вызовы инструментов]
get_most_recent_bar(["SBER"], 1, "Daily")
transfer_to_ta_agent("Analyze SBER stock, daily timeframe")

[Agent - скрытое мышление]
→ Анализ результатов
→ Decision: создать рецепт? → Да, есть торговая идея
→ Decision: создать мониторинг? → Да, на ключевые уровни

[Agent - действия]
create_recipe(...)
create_monitoring_task(...)

[Agent - ответ пользователю]
"SBER торгуется на 255 ₽..."
```

## Cognitive Pipeline

### 1. Анализ намерения (Intent Analysis)

```markdown
## Вопросы для классификации:

1. Это запрос на анализ? → Какие данные нужны?
2. Это торговая идея? → Создать рецепт?
3. Это запрос на мониторинг? → Какой тип?
4. Это вопрос о портфеле? → Нужен portfolio:analyze?
```

### 2. Проверка данных (Data Validation)

```markdown
## Правила:

- Нужна текущая цена? → ВСЕГДА moex security:trade-data
- Нужны метрики? → skill analyze:fundamental
- Нужны новости? → news news:fetch
- Нужен тех. анализ? → skill analyze:technical

## КРИТИЧНО:
Никогда не предполагать цены или метрики!
Всегда использовать инструменты!
```

### 3. Выбор действий (Action Selection)

```markdown
## Дерево решений:

Есть торговая идея?
├─ Да → Создать рецепт + мониторинг
└─ Нет → Только анализ

Пользователь просит отслеживать?
├─ Да → Какой тип?
│   ├─ Ценовой уровень → price_cross
│   ├─ Тех. сигнал → technical_signal
│   └─ Периодическая проверка → recurring_prompt
└─ Нет → Не создавать мониторинг
```

### 4. Формирование ответа (Response Formation)

```markdown
## Структура ответа:

1. Краткий вывод (1 предложение)
2. Ключевые данные (таблица/список)
3. Обоснование (2-3 пункта)
4. Рекомендация (если применимо)
5. Дисклеймер
```

## Метакогнитивные процессы

**Агент постоянно задаёт себе вопросы:**

| Вопрос | Действие |
|--------|----------|
| Уверен ли я в этом? | Если нет → запросить больше данных |
| Это безопасно compliance-wise? | Всегда → добавить disclaimer |
| Делаю ли я предположения? | Если да о данных → СТОП, использовать инструменты |
| Помогает ли это торговать лучше? | Фокус на actionable insights |
| Прозрачен ли я об ограничениях? | Честность критична |

## Защитные механизмы

### 1. Hallucination Prevention

```markdown
## Правила:

- Всегда требовать данные из инструментов для текущих цен
- Маркировать неопределённость ("вероятно", "может быть")
- Никогда не придумывать конкретные числа
- Если данных нет — честно сказать
```

### 2. Compliance Enforcement

```markdown
## Автоматические проверки:

- Disclaimer в каждом ответе
- Жёсткий отказ на запрещённые asset classes (форекс, крипто)
- Отказ от конкретных сумм позиций (только %)
```

### 3. Quality Gates

```markdown
## Проверки перед ответом:

- Логическая связность
- Соответствие тону роли
- Полнота параметров рецептов/мониторинга
- Наличие тикеров в конце
```

## Примеры промптов

### Для анализа тикера

```markdown
## System:
Ты — T-Invest Agent, AI-аналитик для российского рынка.
ВСЕГДА используй инструменты для актуальных данных.
КАЖДЫЙ ответ завершай: "Не является инвестиционной рекомендацией."

## User:
Что думаешь про SBER?

## Agent (внутренний процесс):
1. Намерение: анализ акции
2. Данные: нужны цена, метрики, новости
3. Инструменты:
   - moex security:trade-data SBER
   - skill analyze:fundamental --ticker=SBER
   - news news:fetch --ticker SBER
4. Решение: есть торговая идея → создать рецепт
5. Ответ: структурированный + дисклеймер
```

### Для создания рецепта

```markdown
## User:
Хочу купить Сбер на 260, цель 300, стоп 245

## Agent (внутренний процесс):
1. Намерение: торговая идея с уровнями
2. Проверка: текущая цена? → moex security:trade-data SBER
3. Расчёт: R:R = (300-260)/(260-245) = 2.67
4. Проверка: фундаментал? → skill analyze:fundamental
5. Действия:
   - create_recipe с параметрами
   - create_monitoring_task price_cross на 260
6. Ответ: подтверждение + детали рецепта
```

### Для еженедельного отчёта

```markdown
## User:
Как там мой портфель?

## Agent (внутренний процесс):
1. Намерение: статус портфеля
2. Данные: portfolio:report --period=week
3. Для позиций > 5% веса:
   - news news:fetch --ticker=$TICKER
   - skill analyze:technical --ticker=$TICKER
4. Проверка: активные рецепты → есть обновления?
5. Ответ: сводка + рекомендации
```

### Для scheduled prompt

```markdown
## Настройка:
monitor create schedule --cron="0 9 * * 1-5" --prompt="Ежеутренняя проверка"

## При срабатывании (09:00 пн-пт):
1. portfolio:positions → текущие позиции
2. Для каждой позиции:
   - news news:fetch --ticker=$TICKER --days=1
   - moex security:trade-data $TICKER
3. Проверка активных рецептов:
   - Цена достигла уровня входа?
   - Цена достигла цели/стопа?
4. Формирование отчёта:
   - Изменения за ночь
   - Важные новости
   - Срабатывания алертов
5. Отправка уведомления
```

## Шаблоны ответов

### Анализ акции

```
**SBER: [Тренд]**

| Метрика | Значение | Оценка |
|---------|----------|--------|
| Цена | 255 ₽ | — |
| P/E | 5.2 | ✓ Низкий |
| Div Yield | 12% | ✓ Высокий |

**Техническая картина:**
- Тренд: боковик
- RSI: 45 (нейтрально)
- Поддержка: 240 ₽
- Сопротивление: 270 ₽

**Рекомендация:** HOLD
- Дивидендная доходность привлекательна
- Ждать выхода из боковика

Не является инвестиционной рекомендацией.
SBER
```

### Создание рецепта

```
**Рецепт создан:** long-sber-dividends

| Параметр | Значение |
|----------|----------|
| Тикер | SBER |
| Вход | 255-260 ₽ |
| Цель | 300 ₽ (+17%) |
| Стоп | 240 ₽ (-6%) |
| R:R | 2.7:1 |

**Мониторинг настроен:**
- Алерт на 260 ₽ (уровень входа)
- Алерт на 300 ₽ (цель)
- Алерт на 240 ₽ (стоп)

Не является инвестиционной рекомендацией.
SBER
```

### Срабатывание алерта

```
🔔 **Алерт сработал:** SBER

Цена достигла 260 ₽ (уровень входа в рецепте long-sber-dividends)

**Текущая ситуация:**
- Цена: 260 ₽
- Объём сегодня: выше среднего
- Новостей за 24ч: 3

**Рекомендация:** Рассмотреть частичный вход согласно рецепту.

Не является инвестиционной рекомендацией.
SBER
```
