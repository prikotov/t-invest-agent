# Дашборд Анализа Портфеля

## Задача

Создать комплексный инструмент анализа инвестиционного портфеля, который собирает данные из T-Invest API, анализирует позиции и генерирует красивый интерактивный HTML-дашборд с практическими рекомендациями.

**Все выходные файлы сохраняются в `data/reports/{report_name}/{subname}-YYYYMMDD/`.** Использовать поддиректории по необходимости (`raw/`, `charts/`, `exports/`).

## Доступные Skills

Система имеет доступ к следующим специализированным навыкам. **Загружать и использовать по назначению:**

- **t-invest-portfolio-analysis** — Структура и метрики портфеля из T-Invest
- **t-invest-candles** — Исторические данные OHLCV для теханализа
- **t-invest-orderbook** — Стакан заявок (ликвидность)
- **t-invest-corporate-events** — Календарь дивидендов, отчётностей, событий по облигациям
- **t-invest-trading-calendar** — Торговый календарь и сессии
- **moex-ticker-analysis** — Фундаментальный анализ акций MOEX
- **equity-research** — Профессиональная аналитика (Arsagera, ДоходЪ)
- **social-sentiment** — Сентимент инвесторов с форумов (SmartLab, MFD, Пульс)
- **dashboard** — HTML-шаблоны для интерактивных дашбордов с ECharts графиками (skills/dashboard/SKILL.md)

## Фаза 0: Разведка

Перед глубоким анализом выполнить быструю проверку доступности данных:

1. **Проверить API-соединения** — Убедиться в доступе к T-Invest API, загрузить обзор портфеля
2. **Оценить масштаб портфеля** — Количество позиций, типы активов, общая стоимость
3. **Определить источники данных** — Какие skills нужны для каждого типа позиций
4. **Проверить статус рынка** — Торговые часы, предстоящие события

5. **Спроектировать структуру вывода:**
   ```
   data/reports/{report_name}/{subname}-YYYYMMDD/
   ├── raw/
   │   ├── portfolio.json          # Сырые данные портфеля
   │   ├── positions/              # Данные по позициям
   │   │   ├── SBER.json
   │   │   ├── GAZP.json
   │   │   └── ...
   │   ├── candles/                # Исторические цены
   │   ├── news/                   # Новости и сентимент
   │   └── events/                 # Корпоративные события
   ├── analysis/
   │   ├── structure.json          # Анализ структуры портфеля
   │   ├── risk.json               # Метрики риска
   │   ├── performance.json        # Метрики доходности
   │   └── recommendations.json    # Практические рекомендации
   ├── charts/                     # Данные для графиков
   └── report.html                 # Итоговый интерактивный дашборд
   ```

**Ключевые принципы:**
- **Атомарная запись** — Сохранять инкрементально, не терять прогресс при прерывании
- **Возобновляемость** — При прерывании пропускать уже собранные данные
- **Свежие данные** — Всегда получать актуальные цены и метрики (без кэша)
- **Разделение** — Сырые данные отдельно от анализа

Результаты разведки сохранить в `DESIGN_NOTES.md` в директории отчёта.

## Фаза 1: Сбор данных

### 1.1 Снимок портфеля

Используя skill **t-invest-portfolio-analysis**:
- Общая стоимость портфеля (текущая vs. предыдущие периоды)
- Все позиции: тикер, название, количество, средняя цена, текущая цена, P&L
- Распределение по типам активов (акции, облигации, фонды, валюты)
- Разбивка по секторам/странам (если доступно)

Сохранить в `raw/portfolio.json`.

### 1.2 Глубокий анализ позиций

Для каждой значимой позиции (>1% портфеля):

**Ценовые данные (t-invest-candles):**
- Дневные свечи за 30 дней (краткосрочные тренды)
- Недельные свечи за 12 месяцев (долгосрочные тренды)
- Анализ объёмов

**Фундаментальные данные (moex-ticker-analysis):**
- Мультипликаторы P/E, P/B, P/S
- Дивидендная доходность и история
- Рост выручки/EPS
- Уровень долга

**Корпоративные события (t-invest-corporate-events):**
- Предстоящие дивиденды
- Даты отчётностей
- Другие существенные события

**Исследования (equity-research):**
- Мнения профессиональных аналитиков
- Целевые цены
- Инвестиционный тезис

**Сентимент (social-sentiment):**
- Недавние обсуждения на форумах
- Индикаторы бычьего/медвежьего настроя
- Ключевые темы и катализаторы

Данные по каждой позиции сохранить в `raw/positions/{TICKER}.json`.

### 1.3 Рыночный контекст

- **Торговый календарь (t-invest-trading-calendar):** Предстоящие праздники, специальные сессии
- **Стаканы (t-invest-orderbook):** Проверка ликвидности для крупных позиций
- **Общие рыночные новости:** Широкие рыночные условия, влияющие на портфель

### 1.4 Анонимизация позиций (опционально)

Сгенерировать файл маппинга, который преобразует реальные тикеры в **обобщённые псевдонимы** — категории без раскрытия конкретных компаний. Примеры:

- `SBER` → `Крупнейший банк РФ`
- `GAZP` → `Нефтегазовый гигант`
- `LKOH` → `Нефтяная компания`
- `YNDX` → `Технологическая компания`
- `GMKN` → `Металлургический холдинг`

Использовать сектор, размер компании и профиль деятельности для определения псевдонима. Длина: 2-4 слова.

**Важно:** Дашборд должен иметь переключатель: "Показать реальные тикеры" / "Показать анонимизированные". По умолчанию — реальные. Это позволяет шерить отчёт без раскрытия состава портфеля.

Сохранить маппинг в `raw/anonymization.json`.

## Фаза 2: Анализ

### 2.1 Анализ структуры портфеля

Рассчитать и проанализировать:
- **Риск концентрации**: Топ-5 позиций в % портфеля, индекс Херфиндаля
- **Секторальное распределение**: Разбивка по отраслям
- **Географическое распределение**: Россия vs. международные рынки
- **Смесь типов активов**: Акции, облигации, кэш, прочее
- **Валютная экспозиция**: Позиции в RUB, USD, EUR

Сохранить в `analysis/structure.json`.

### 2.2 Метрики риска

Рассчитать:
- **Волатильность**: На уровне портфеля и по позициям (из исторических данных)
- **Просадка**: Максимальная просадка от пика
- **Бета**: Корреляция с рыночным индексом (IMOEX)
- **Value at Risk (VaR)**: 1-дневный и 1-недельный при 95% доверии
- **Матрица корреляций**: Как позиции движутся вместе

Сохранить в `analysis/risk.json`.

### 2.3 Анализ доходности

Рассчитать:
- **Общая доходность**: С начала, YTD, последние 30/90/365 дней
- **Сравнение с бенчмарком**: vs. IMOEX, vs. инфляция
- **Лучшие/худшие позиции**: Топ прибыльных и убыточных
- **Дивидендный доход**: Доходность по портфелю, предстоящие дивиденды
- **Атрибуция**: Что обеспечило доходность (цена vs. дивиденды)
- **Паттерны по дням недели**: В какой день недели портфель чаще растёт/падает
- **Лучший/худший месяц**: Статистика по месяцам

Сохранить в `analysis/performance.json`.

### 2.4 Движок рекомендаций

Сгенерировать практические рекомендации:
- **Предложения по ребалансировке**: Перевешенные/недовешенные позиции
- **Риск-алерты**: Концентрированные позиции, высокая корреляция
- **Возможности**: Недооценённые позиции, позитивные сдвиги сентимента
- **Требующие внимания**: Предстоящие события (дивиденды, отчётности)

Для каждой рекомендации указать:
- Тикер
- Действие: ПОКУПКА / ПРОДАЖА / ДЕРЖАТЬ / НАБЛЮДЕНИЕ
- Обоснование (2-3 предложения)
- Уровень уверенности
- Временной горизонт

Сохранить в `analysis/recommendations.json`.

## Фаза 3: Визуализации (HTML Дашборд)

### Цели дизайна

Создать одностраничный HTML-файл с встроенными CSS и JS:
- **Тёмная тема** по умолчанию (переключатель на светлую)
- **Адаптивный layout** (работает на десктопе и планшете)
- **Плавные анимации** при загрузке
- **Чистая типографика** (Inter или системные шрифты)
- **Премиальный вид** — ощущение профессиональной инвестиционной аналитики

### Рекомендуемые шаблоны

**Использовать готовые шаблоны из `skills/dashboard/templates/`:**

- **portfolio.html** — Базовый шаблон для дашборда портфеля с ECharts графиками
- **stock-details.html** — Детальная страница акции (для drill-down модалов)
- **watchlist.html** — Вотчлист с отслеживанием позиций
- **stock-dashboard.html** — Общий дашборд рынка

**Преимущества:**
- Phoenix Theme с переключателем light/dark
- ECharts уже настроен (автоинициализация по CSS-классам)
- Адаптивный layout, работает офлайн (file:// protocol)
- Единый стиль визуализаций

**Workflow:**
1. Взять подходящий шаблон из `skills/dashboard/templates/`
2. Встроить собранные данные в JavaScript переменные
3. Настроить графики под конкретный портфель
4. Сохранить результат в `data/reports/{report_name}/{subname}-YYYYMMDD/report.html`

При отсутствии подходящего шаблона — создать на основе Phoenix Theme, используя Chart.js или D3.js через CDN.

### Обязательные секции

#### 3.1 Баннер ключевых метрик

Карточки сверху с ключевыми показателями:
- Общая стоимость портфеля
- Общий P&L (абсолютный и в %)
- Изменение за день
- Количество позиций
- Бета портфеля
- Коэффициент Шарпа (если рассчитывается)

#### 3.2 Распределение активов

- **Кольцевая диаграмма**: Разбивка по типам активов (акции, облигации, фонды, кэш)
- **Treemap или столбчатая диаграмма**: Распределение по секторам
- **Диаграмма валютной экспозиции**

#### 3.3 Тепловая карта доходности (GitHub-style)

- Полная календарная тепловая карта за последние 12 месяцев
- Цвет ячейки = доходность портфеля в этот день (зелёный = прибыль, красный = убыток)
- Hover: показать дату, доходность, ключевые события
- Возможность прокрутки к предыдущим годам
- Палитра: тёмный фон → зелёный градиент (или красный для убытков)

#### 3.4 Временная шкала доходности

- **Линейный график**: Стоимость портфеля во времени (последние 12 месяцев)
- **Area chart**: Накопленный P&L с наложением бенчмарка
- **Столбчатая диаграмма месячной доходности**

#### 3.4 Таблица анализа позиций

Интерактивная таблица со всеми позициями:
- Тикер, Название
- Количество, Средняя цена, Текущая цена
- P&L (абсолютный и в %)
- Вес в портфеле
- Изменение за 30 дней
- Быстрые действия (клик для детального просмотра)

Функции:
- Сортируемые столбцы
- Фильтр по типу актива, сектору
- Цветовая кодировка прибыли/убытка

#### 3.5 Дашборд риска

- **Индикатор риска концентрации**: Визуальный показатель (Низкий/Средний/Высокий)
- **Тепловая карта корреляций**: Между топ-10 позициями
- **Сравнение волатильности**: Портфель vs. бенчмарк
- **График просадок**: Исторические просадки

#### 3.6 Тепловая карта по дням недели

- Сетка: X = день месяца (1-31) или час торгов (10-18), Y = день недели (Пн-Пт)
- Цвет = средняя доходность в этот период
- Показывает паттерны: "Понедельники обычно красные", "После обеда больше волатильности"

#### 3.7 Дивидендный календарь

Временная шкала с:
- Предстоящие даты дивидендов
- Ожидаемые суммы
- Выделенные даты отсечки

#### 3.8 Обзор сентимента

Для крупных позиций (>5% портфеля):
- Индикатор сентимента (Бычий/Нейтральный/Медвежий)
- Недавний объём обсуждений
- Ключевые темы

#### 3.9 Панель рекомендаций

Список практических рекомендаций:
- Цветовая кодировка по действию (зелёный=покупка, красный=продажа, жёлтый=наблюдение)
- Ранг приоритета
- Клик для раскрытия обоснования

#### 3.10 Вотчлист

Позиции для мониторинга:
- Приближение к ключевым уровням
- Предстоящие катализаторы
- Сдвиги сентимента

#### 3.11 Интересные инсайты

Секция с неочевидными и персонализированными находками, сгенерированными из данных. Примеры:

- "Твой самый прибыльный день недели: четверг 📈 (+2.3% в среднем)"
- "Самая долгая позиция: SBER держишь 245 дней"
- "Лучшая сделка года: +47% на GAZP за 23 дня"
- "Дивидендный король: SBER принёс 12,500 ₽ дивидендами"
- "Ты пережил максимальную просадку -18% в марте, но не продал 💪"
- "Рекорд диверсификации: 8 секторов в портфеле"
- "Самый волатильный актив: POLY (σ=4.2%)"
- "Топ-3 слов в сентименте: 'рост', 'дивиденды', 'отчётность'"

Быть креативным. Использовать эмодзи. Чем интереснее и персональнее инсайты, тем лучше.

### Интерактивность

- **Селектор периода**: 7Д / 1М / 3М / 6М / 1Г / Все
- **Переключатель анонимизации**: "Реальные тикеры" / "Анонимизированные" (для шаринга)
- **Drill-down по позиции**: Клик → детальный модал с:
  - Полные фундаментальные данные
  - График цены с объёмом
  - Новости и сентимент
  - Мнения аналитиков
- **Кнопка экспорта**: Скачать данные как CSV/JSON
