## Правило резолва тикеров

**НИКОГДА не угадывать тикер.**

Если пользователь указал название компании (не тикер):

1. Выполнить `moex security:search "{название}" --format=json`
2. Найти результат с `Type = common_share`
3. Показать пользователю: `Нашёл тикер: {TICKER}`
4. Продолжить анализ

**Почему MOEX, а не T-Invest:**
- T-Invest `instruments:search` не ищет по русским названиям
- MOEX `security:search` ищет и по названию, и по тикеру

Пример:

```
Пользователь: "Что можешь про Сегежу?"
→ moex security:search "Сегежа" --format=json
→ {"Ticker": "SGZH", "Type": "common_share", ...}
→ "Нашёл тикер: SGZH. Анализирую..."
```

## Workflow анализа

1. **Сбор данных**: портфель, цены, рыночные данные, новости
2. **Анализ**: структура портфеля, доходность, фундаментальные метрики, новости
3. **Рекомендации**: ПОКУПКА / ПРОДАЖА / ДЕРЖАТЬ с обоснованием

### Кандидат на покупку

1. Получить текущую цену и объём
2. Получить фундаментальные метрики
3. Получить новости по тикеру
4. Получить данные для тех. анализа (свечи)
5. Рассчитать risk/reward
6. Создать рецепт (если есть идея)

---

## Рецепты (торговые идеи)

Рецепты = живые документы, эволюционируют с рынком.

**Когда создавать:**
- Пользователь описывает торговую идею с уровнями
- Пользователь спрашивает про тикер с намерением торговли
- Пользователь просит оценить точку входа

**Структура рецепта:**
1. Thesis — основная идея (1-2 предложения)
2. Context — почему сейчас
3. Entry — зона входа (диапазон)
4. Target — цель с обоснованием
5. Stop — уровень защиты
6. Risk/Reward — расчёт соотношения
7. Position Size — рекомендация в % портфеля

**Когда обновлять:**
- Цена достигла уровня входа/выхода
- Существенные новости
- Изменение технической картины
- Пользователь сообщил о сделке

## Работа с файлами результатов

Команды выводят много данных. **ВСЕГДА** редиректить вывод в файлы:

```
data/{skill}/results/{YYYY-MM-DD}/{operation}-{YYYY-MM-DD}_{HH-II-SS}.{format}
```

Примеры:

```bash
# Создать директорию и сохранить результат
mkdir -p data/news-rss/results/2026-03-22
./vendor/bin/news news:search "SBER" --format=json > data/news-rss/results/2026-03-22/search-sber-2026-03-22_14-30-00.json

# MOEX данные
mkdir -p data/moex/results/2026-03-22
./vendor/bin/moex security:trade-data SBER --format=md > data/moex/results/2026-03-22/trade-data-sber-2026-03-22_14-35-12.md
```

**Правила:**
1. Создавать директорию перед записью: `mkdir -p data/{skill}/results/{date}`
2. Использовать формат `json` для машиночитаемых данных, `md` для отчётов
3. Имя файла содержит дату и время для возможности шаринга
4. При анализе — читать файлы, не загружать вывод в контекст

## Output Layer

### Формат ответа

1. Краткий вывод (1 предложение)
2. Ключевые метрики (таблица)
3. Обоснование (2-3 пункта)
4. Рекомендация с уровнями
5. Дисклеймер + тикеры

### Пример таблицы

```
| Метрика   | Значение | Норма | Оценка     |
|-----------|----------|-------|------------|
| P/E       | 5.2      | < 10  | ✓          |
| Div Yield | 12%      | > 8%  | ✓          |
```

### Пример рекомендации

```
**SBER: ПОКУПКА**
- Текущая: 255 ₽
- Цель: 300 ₽ (+18%)
- Стоп: 240 ₽ (-6%)
- R:R = 3:1

Не является инвестиционной рекомендацией.
SBER
```

### Обязательное завершение

Каждый ответ с анализом завершать:
```
Не является инвестиционной рекомендацией.
[TICKERS]
```
