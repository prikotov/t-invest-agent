# News Web Search

Поиск новостей по новостным сайтам для анализа новостного фона компании.

**Источники:**
- **Interfax** — основной, работает через webfetch
- **Kommersant** — деловые новости, работает через webfetch  
- **RIA** — API endpoint, работает через webfetch
- **PRIME** — экономические новости, API endpoint
- **TASS** — требует headless Chrome (JS-челлендж)
- **RBC** — недоступен (блокирует curl и headless Chrome, сильная защита от ботов)

## Прямой доступ (webfetch)

| Источник | URL | Особенности |
|----------|-----|-------------|
| Interfax | `https://www.interfax.ru/search/?df={from}&dt={to}&phrase={query}` | Глубокий архив, поиск по всем разделам |
| Kommersant | `https://www.kommersant.ru/Search/Results?search_query={query}` | Деловые новости |
| RIA | `https://ria.ru/services/search/getmore/?query={query}&tags_limit=20&interval%5B%5D=year&sort%5B%5D=date` | API возвращает JSON-подобный формат |
| PRIME | `https://1prime.ru/services/search/getmore/?tags_limit=20&date_from={from}&date_to={to}&query={query}` | API с датами в формате YYYY-MM-DD |

## Только через браузер (требуют JS-челлендж)

| Источник | URL | Особенности |
|----------|-----|-------------|
| TASS | `https://tass.ru/search?search={query}&from_date={from}&to_date={to}` | Даты в ISO формате с Z: `2026-02-28T17:00:00.000Z` |

**TASS через headless Chrome:**

```bash
google-chrome --headless=new --disable-gpu --disable-blink-features=AutomationControlled \
  --window-size=1920,1080 \
  --user-agent="Mozilla/5.0 (X11; Linux x86_64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/146.0.0.0 Safari/537.36" \
  --dump-dom --timeout=30000 \
  'https://tass.ru/search?search={QUERY}&from_date={FROM}&to_date={TO}' > tass.html
```

Параметры:
| Параметр | Описание |
|----------|----------|
| search | Поисковая фраза (URL-encoded) |
| from_date | Дата начала (ISO: `2026-02-28T17:00:00.000Z`) |
| to_date | Дата окончания (ISO: `2026-03-20T16:59:59.000Z`) |

Извлечение заголовков из HTML:
```bash
grep -oP 'MaterialCardLayout_text__LnYk4">\K[^<]+' tass.html
grep -oP 'NonMediaMaterialCardLayout_text__nc9M6">\K[^<]+' tass.html
```

## Специализация источников

| Источник | Сила | Тип контента |
|----------|------|--------------|
| **Interfax** | Финансы, назначения, облигации | Прямые новости компании, пресс-релизы |
| **Kommersant** | Отраслевая аналитика, интервью | Бизнес-статьи, комментарии экспертов |
| **PRIME** | Экономика, макро | Рыночные обзоры, макроэкономика |
| **RIA** | Общие новости | Краткие новости, широкое покрытие |

**Рекомендация:** Искать параллельно по Interfax + Kommersant для полного охвата.

## Типология новостей

При анализе результатов разделять:

| Тип | Пример | Ценность |
|-----|--------|----------|
| **Прямые** | "Segezha разместила облигации" | Высокая — конкретные факты |
| **Отраслевые** | "Рынок гофропродукции в 2026" | Средняя — контекст |
| **Сводки** | "Индекс МосБиржи просел, в т.ч. SGZH" | Низкая — шум |

**Фильтрация шума:**
- Interfax: игнорировать заголовки "Индекс МосБиржи...", "Рынок акций..."
- PRIME: игнорировать "Российский рынок акций..."
- Оставлять только новости где компания — **субъект**, не упоминание

## Как использовать

### Шаг 1: Определить поисковые термины

По тикеру определить название компании и варианты:

| Тикер | Название | Поисковые термины |
|-------|----------|-------------------|
| SBER | Сбербанк | "Сбербанк", "Sber", "Греф" |
| GAZP | Газпром | "Газпром", "Gazprom", "Миллер" |
| LKOH | Лукойл | "Лукойл", "Lukoil", "Алекперов" |
| YNDX | Яндекс | "Яндекс", "Yandex" |
| TCSG | Т-Банк | "Тинькофф", "Т-Банк", "Tinkoff" |
| SGZH | Сегежа | "Сегежа", "Segezha" |

### Шаг 2: Алгоритм поиска (КРИТИЧНО)

**Принцип: от свежего к глубокому. Ищем каскадно, расширяя период если нет значимых новостей.**

```
Этап 1: Последний месяц
  → Если есть 3+ значимых новости: СТОП, выводим результат
  
Этап 2: Последние 3 месяца (если < 3 значимых на этапе 1)
  → Если есть 3+ значимых новости: СТОП, выводим результат
  
Этап 3: Последние 6 месяцев (если < 3 значимых на этапе 2)
  → Выводим все найденные значимые новости
```

**Значимая новость = компания субъект действия:**
- `[Компания] разместила облигации` — да
- `[Компания] назначила CFO` — да
- `[Компания] отчиталась за квартал` — да
- `Индекс МосБиржи просел, в т.ч. [компания]` — НЕТ (шум)
- `Рынок [отрасль] в 2026 году` — да (если есть комментарий компании)

**Параллельный запрос на каждом этапе:**

```
// ОДИН блок tool calls:
- webfetch Interfax (период этапа)
- webfetch Kommersant (без ограничения даты)
- webfetch PRIME (период этапа)
```

**Почему параллельно:**
- Interfax силён в финансовых новостях (облигации, CFO, дивиденды)
- Kommersant силён в отраслевых статьях и интервью
- Один источник может не иметь нужной информации

### Шаг 3: Поиск через Interfax

**URL формат:**
```
https://www.interfax.ru/search/?df={FROM}&dt={TO}&phrase={QUERY}
```

Параметры:
| Параметр | Формат | Пример |
|----------|--------|--------|
| df | DD.MM.YYYY | `01.01.2026` |
| dt | DD.MM.YYYY | `22.03.2026` |
| phrase | URL-encoded | `%D0%A1%D0%B5%D0%B3%D0%B5%D0%B6%D0%B0` |

**Извлечение ссылок:**

Формат ссылки в результатах:
```
https://www.interfax.ru/business/1043814
https://www.interfax.ru/russia/1051845
```

Полный URL = `https://www.interfax.ru` + путь из атрибута href

**Фильтрация шума Interfax:**

Игнорировать заголовки с паттернами:
- `Индекс МосБиржи...`
- `Рынок акций...`
- `Российский рынок...`

Оставлять заголовки где компания — **субъект действия**:
- `[Компания] разместила...`
- `[Компания] назначила...`
- `[Компания] отчиталась...`

### Шаг 4: Поиск через Kommersant

```
https://www.kommersant.ru/Search/Results?search_query={QUERY}
```

**Сила Kommersant:**
- Отраслевые статьи (рынок, тренды)
- Интервью с руководством
- Бизнес-аналитика
- Эксклюзивные материалы

**Извлечение ссылок:**

Формат ссылки в результатах:
```
https://www.kommersant.ru/doc/8400650
https://www.kommersant.ru/doc/8234973
```

Полный URL = `https://www.kommersant.ru/doc/{ID}` из атрибута href

### Шаг 5: Поиск через RIA

```
https://ria.ru/services/search/getmore/?query={QUERY}&tags_limit=20&interval%5B%5D=year&sort%5B%5D=date
```

Параметры:
| Параметр | Описание |
|----------|----------|
| query | Поисковая фраза (URL-encoded) |
| tags_limit | Лимит результатов |
| interval[] | Период: `year`, `month`, `week`, `all` |
| sort[] | Сортировка: `date`, `relevance` |

Примеры:
```
https://ria.ru/services/search/getmore/?query=%D0%A1%D0%B5%D0%B3%D0%B5%D0%B6%D0%B0&tags_limit=20&interval%5B%5D=year&sort%5B%5D=date
https://ria.ru/services/search/getmore/?query=%D0%A1%D0%B1%D0%B5%D1%80%D0%B1%D0%B0%D0%BD%D0%BA&tags_limit=50&interval%5B%5D=all&sort%5B%5D=date
```

### Шаг 6: Поиск через PRIME

```
https://1prime.ru/services/search/getmore/?tags_limit=20&date_from={FROM}&date_to={TO}&query={QUERY}
```

Параметры:
| Параметр | Описание |
|----------|----------|
| query | Поисковая фраза (URL-encoded) |
| tags_limit | Лимит результатов |
| date_from | Дата начала (YYYY-MM-DD) |
| date_to | Дата окончания (YYYY-MM-DD) |

Примеры:
```
https://1prime.ru/services/search/getmore/?tags_limit=20&date_from=2026-03-01&date_to=2026-03-21&query=%D0%A1%D0%B5%D0%B3%D0%B5%D0%B6%D0%B0
https://1prime.ru/services/search/getmore/?tags_limit=50&date_from=2025-01-01&date_to=2026-03-22&query=%D0%A1%D0%B1%D0%B5%D1%80%D0%B1%D0%B0%D0%BD%D0%BA
```

### Шаг 7: Чтение полных статей

Для релевантных статей открыть полный текст по ссылке и извлечь:
- Дату публикации
- Суть новости
- Заявления руководства
- Финансовые прогнозы
- Связанные события

**Приоритет открытия:**
1. Прямые новости о компании (облигации, назначения, отчётность)
2. Отраслевые статьи с комментариями компании
3. Интервью с руководством

### Шаг 8: Агрегация и дедупликация

**После получения результатов от всех источников:**

1. **Объединить** новости по дате
2. **Дедуплицировать** — одно событие может быть в нескольких источниках
3. **Фильтровать** — убрать сводки рынка где компания лишь упомянута
4. **Группировать** по категориям

**Формат вывода (с ссылками на источники):**

```
## [TICKER] — Новости за [период]

| Дата | Новость | Источник |
|------|---------|----------|
| DD.MM.YYYY | Заголовок новости | [Interfax](URL) |
| DD.MM.YYYY | Заголовок новости | [Kommersant](URL) |

### Ключевые темы:
- Тема 1: описание
- Тема 2: описание
```

**Обязательно:**
- Каждая новость должна иметь кликабельную ссылку на источник
- Формат ссылки: `[Название источника](URL)`
- URL должен быть полным (с https://)

### Шаг 9: Структурировать результаты

Сгруппировать новости по темам:

| Категория | Что искать |
|-----------|------------|
| Стратегия | Планы развития, новые направления |
| Финансы | Отчётность, дивиденды, долговая нагрузка |
| Риск-факторы | Суды, санкции, конфликты, проверки |
| Руководство | Назначения, заявления, интервью |
| Сделки | M&A, продажи активов, партнёрства |

## Типовые сценарии

### Быстрый поиск свежих новостей (по умолчанию)

```
Этап 1 (параллельно):
- Interfax: последние 30 дней
- Kommersant: общий поиск
- PRIME: последние 30 дней

→ Если >= 3 значимых новостей: СТОП, вывод с ссылками

Этап 2 (если < 3 значимых):
- Interfax: последние 90 дней
- PRIME: последние 90 дней

→ Если >= 3 значимых новостей: СТОП, вывод с ссылками

Этап 3 (если < 3 значимых):
- Interfax: последние 180 дней
- PRIME: последние 180 дней

→ Вывод всех найденных значимых новостей с ссылками
```

### Анализ новой компании

```
// Параллельно:
1. Interfax: df=01.01.2024&dt=сегодня&phrase=Название
2. Kommersant: название компании
3. PRIME: последние 12 месяцев

// Извлечь: стратегия, планы, риски, финансовое состояние
```

### Проверка заявлений CEO

```
1. Interfax: phrase=Фамилия%20CEO&df=полгода&dt=сегодня
2. Kommersant: "Фамилия CEO" (интервью)
3. Сверить с реальными результатами
```

### Поиск риск-факторов

```
1. Interfax: phrase=компания%20(суд|проверка|скандал)
2. Kommersant: "компания" + (суд|проверка|скандал)
3. Оценить серьёзность и текущий статус
```

### Дивидендная история

```
1. Interfax: phrase=компания%20дивиденды&df=5%20лет&dt=сегодня
2. Извлечь: заявленные vs выплаченные
```

## Типичные ошибки

| Ошибка | Пример | Решение |
|--------|--------|---------|
| Последовательный поиск | Сначала Interfax, потом Kommersant | Параллельно в одном блоке tool calls |
| Игнорирование шума | Включать все сводки рынка | Фильтровать по типу новости |
| Один источник | Только Interfax | Минимум 2 источника параллельно |
| Широкий период сразу | 6 месяцев при поиске свежих новостей | Каскад: месяц → 3 месяца → 6 месяцев |
| Нет ссылок | Только заголовки без URL | Обязательно `[Источник](URL)` в таблице |

## Результат

Структурированная аналитика:

```
## [COMPANY] — Исторический обзор

### Стратегия и планы
- [Дата] План X → Статус: выполнен/нет
- [Дата] Заявление Y → Результат

### Риск-факторы
- [Дата] Событие → Исход

### Тренд заявлений руководства
- Оптимизм/Реализм → Тенденция

### Вывод
[Сводка по истории компании]
```

## Ограничения

| Источник | Доступность | Особенности |
|----------|-------------|-------------|
| Interfax | webfetch | Много шума в результатах (рыночные сводки) |
| Kommersant | webfetch | Хорошая релевантность, отраслевые статьи |
| PRIME | webfetch | API формат, экономические новости |
| RIA | webfetch | Общие новости, широкий охват |
| TASS | headless Chrome | Требует JS-челлендж |
| RBC | недоступен | Блокирует автоматические запросы |

**Ключевые ограничения:**
- Interfax выдаёт 50+ результатов, 80% — шум (сводки рынка)
- Нужно фильтровать по заголовкам
- Параллельный поиск обязателен для полноты картины
