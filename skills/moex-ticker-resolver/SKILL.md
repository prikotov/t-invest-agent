# MOEX Ticker Resolver

## Когда использовать

- Пользователь называет компанию по имени (Сегежа, Сбербанк, Газпром)
- Тикер неизвестен или нужно проверить
- Перед анализом акции, если тикер не указан явно

## Ограничения

**НИКОГДА не угадывать тикер.** T-Invest API не ищет по русским названиям — только MOEX API.

## Как использовать

**Шаг 1:** Поиск инструмента по названию

```bash
./vendor/bin/moex security:search "{название}" --format=json
```

Аргументы:

| Аргумент | Описание                    |
|----------|-----------------------------|
| query    | Название компании или тикер |

Опции:

| Опция    | Описание       | Значения            | По умолчанию |
|----------|----------------|---------------------|--------------|
| --format | Формат вывода  | table, json, csv, md | table        |

Примеры:

```bash
./vendor/bin/moex security:search "Сегежа" --format=json
./vendor/bin/moex security:search "Газпром" --format=json
./vendor/bin/moex security:search "Сбер" --format=json
```

**Шаг 2:** Выбрать акцию из результатов

Из ответа выбрать инструмент где:
- `Type` = `common_share` (обыкновенная акция)
- `Group` = `stock_shares`

Если несколько результатов — выбрать с `Type = common_share`.

**Шаг 3:** Показать тикер пользователю

Перед продолжением анализа сообщить найденный тикер:

```
Нашёл тикер: SGZH (Сегежа)
```

## Результат

| Поле  | Описание                |
|-------|-------------------------|
| Ticker| Тикер инструмента       |
| Name  | Название компании       |
| Type  | Тип инструмента         |
| Group | Группа инструментов     |

## Пример workflow

```
Пользователь: "Что можешь рассказать про Сегежу?"

1. moex security:search "Сегежа" --format=json
2. Результат: [{"Ticker": "SGZH", "Name": "Сегежа", "Type": "common_share", ...}]
3. Ответ: "Нашёл тикер: SGZH. Продолжаю анализ..."
4. Продолжить с moex-ticker-analysis skill
```

## Связанные skills

- `moex-ticker-analysis` — анализ акции после определения тикера
- `moex` — спецификации и рыночные данные
